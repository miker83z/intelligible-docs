Intelligible Identity and Certificate SDK

The creation of identities and certificates through the Intelligible paradigm.

This SDK includes the software that creates identidies and certificates based on two models:

  1. Intelligible Identity
  2. Intelligible Certificate

These two modules bring together the ability of Distributed Ledger Technologies to provide a guarantee on digital documents validation and the ability for machines to extract knowledge from documents.

Both models are built on the use of a Non Fungible Token, implemented as a ERC721 Token in Ethereum, and on the use of common data structures represented in XML to provide metadata.

The implementation presented here consists of four modules:

  • metadata-doc - used to create and parse the metadata XML document
  • intelligible-token-ethereum - used to manage the interaction with the ERC721 tokens
  • intelligible-identity - implementation of the Intelligible Identity model; depends on the previous two modules
  • intelligible-certificate - implementation of the Intelligible Certificate model; depends on the previous three modules
